# React — 엘리먼트와 컴포넌트의 종류: Host, Custom, Static

## 1. React 엘리먼트란?

### 1.1 정의

React 엘리먼트는 **UI의 한 시점을 표현하는 순수 자바스크립트 객체**이다.

```javascript
// JSX
const element = <button onClick={handleClick}>클릭</button>;

// 컴파일 후 React.createElement 호출 결과
const element = {
  $$typeof: Symbol(react.element),
  type: "button", // 어떤 컴포넌트/태그인가
  key: null,
  ref: null,
  props: {
    onClick: handleClick,
    children: "클릭",
  },
};
```

React 엘리먼트는 **불변(immutable)** 이다. 한 번 생성되면 변경되지 않는다.  
UI의 변경은 새로운 엘리먼트를 생성하여 기존 것과 교체하는 방식으로 이루어진다.

### 1.2 React 엘리먼트의 `type` 필드

React 엘리먼트의 종류는 `type` 필드 값으로 구분한다.

| `type` 값                    | 종류              | 예시                         |
| ---------------------------- | ----------------- | ---------------------------- |
| 문자열 (`'div'`, `'button'`) | DOM 엘리먼트      | `<div>`, `<input>`           |
| 함수 또는 클래스             | 컴포넌트 엘리먼트 | `<MyComponent />`            |
| `Symbol(react.fragment)`     | Fragment          | `<>...</>`                   |
| `Symbol(react.lazy)`         | Lazy              | `React.lazy(...)`            |
| `Symbol(react.memo)`         | Memo              | `React.memo(...)`            |
| `Symbol(react.portal)`       | Portal            | `ReactDOM.createPortal(...)` |
| `Symbol(react.suspense)`     | Suspense          | `<Suspense>`                 |

## 2. 컴포넌트의 세 가지 분류

React 까보기 시리즈에서는 컴포넌트를 내부 동작 방식에 따라 세 가지로 분류한다.

### 2.1 Host 컴포넌트

**브라우저(호스트 환경)가 직접 이해할 수 있는 DOM 태그**에 대응하는 컴포넌트이다.

```jsx
// Host 컴포넌트가 반환하는 엘리먼트들
function MyComponent() {
  return (
    <div>
      {" "}
      {/* Host 컴포넌트 */}
      <button>
        {" "}
        {/* Host 컴포넌트 */}
        <input /> {/* Host 컴포넌트 */}
      </button>
    </div>
  );
}
```

- `type` 필드가 **문자열**('div', 'button', 'input' 등)인 엘리먼트
- Reconciler가 이 엘리먼트를 처리할 때 **실제 DOM 노드를 생성**한다
- Fiber 노드의 `tag` 값: `HostComponent` (5)

### 2.2 Custom 컴포넌트 (사용자 정의 컴포넌트)

**개발자가 직접 선언한** 함수형 컴포넌트 또는 클래스형 컴포넌트이다.

```jsx
// 함수형 Custom 컴포넌트
function UserCard({ name }) {
  return <div>{name}</div>;
}

// 클래스형 Custom 컴포넌트
class UserCard extends React.Component {
  render() {
    return <div>{this.props.name}</div>;
  }
}

// 사용 시 생성되는 엘리먼트
const element = <UserCard name="Alice" />;
// element.type = UserCard (함수 또는 클래스 자체)
```

- `type` 필드가 **함수 또는 클래스**인 엘리먼트
- Reconciler는 이 엘리먼트를 만나면 **컴포넌트 함수를 호출**하여 반환값을 다시 처리한다
- 함수형 Fiber 노드 `tag`: `FunctionComponent` (0)
- 클래스형 Fiber 노드 `tag`: `ClassComponent` (1)

> ⚠️ **Custom 컴포넌트와 Host 컴포넌트의 처리 차이:**  
> Host 컴포넌트(`<div>`) → Reconciler가 직접 DOM 노드 생성  
> Custom 컴포넌트(`<UserCard>`) → Reconciler가 컴포넌트 함수를 **호출**하여 반환값(React 엘리먼트)을 다시 처리

### 2.3 Static 컴포넌트 (React 내장 컴포넌트)

React가 미리 선언해 둔 특수 목적 컴포넌트들이다.

| Static 컴포넌트 | 사용 방식                            | 역할                          |
| --------------- | ------------------------------------ | ----------------------------- |
| `Fragment`      | `<>...</>` 또는 `<React.Fragment>`   | DOM 노드 없이 여러 자식 묶기  |
| `Suspense`      | `<Suspense fallback={<Loading />}>`  | 비동기 컴포넌트 로딩 대기     |
| `Lazy`          | `React.lazy(() => import('./Comp'))` | 코드 스플리팅, 지연 로딩      |
| `Memo`          | `React.memo(Component)`              | props 변경 없으면 리렌더 생략 |
| `Portal`        | `ReactDOM.createPortal(...)`         | DOM 트리 외부에 렌더링        |
| `StrictMode`    | `<React.StrictMode>`                 | 개발 모드 추가 검사           |

> ⚠️ **원문 오류 정정**: 원문에서 "스태틱 컴포넌트"에 "텍스트"를 포함했지만, 텍스트 노드는 React 엘리먼트가 아닌 **문자열 또는 숫자** 타입의 `children`으로 처리된다. 별도의 컴포넌트가 아니다.

## 3. 컴포넌트와 엘리먼트의 관계

```
컴포넌트 (Component)
  = props를 받아 React 엘리먼트를 반환하는 함수 (또는 클래스)
  ≠ React 엘리먼트 자체

React 엘리먼트 (Element)
  = 컴포넌트가 반환하는 자바스크립트 객체
  = UI의 한 시점에 대한 설명서
```

```jsx
// Button은 컴포넌트
function Button({ label }) {
  return <button>{label}</button>; // React 엘리먼트 반환
}

// <Button label="클릭" />은 React 엘리먼트 (컴포넌트 엘리먼트)
// { type: Button, props: { label: '클릭' } }

// <button>클릭</button>은 React 엘리먼트 (DOM 엘리먼트)
// { type: 'button', props: { children: '클릭' } }
```

## 4. Reconciler가 각 컴포넌트 타입을 처리하는 방식

```
Reconciler가 React 엘리먼트를 Fiber로 변환할 때:

element.type이 문자열?
  → HostComponent Fiber 생성
  → 커밋 페이즈에서 실제 DOM 노드 생성

element.type이 함수?
  → FunctionComponent Fiber 생성
  → renderWithHooks로 컴포넌트 함수 호출
  → 반환된 엘리먼트를 다시 처리

element.type이 클래스?
  → ClassComponent Fiber 생성
  → 인스턴스 생성 → render() 호출
  → 반환된 엘리먼트를 다시 처리

element.type이 Symbol(react.fragment)?
  → Fragment Fiber 생성
  → DOM 노드 없이 자식만 처리

element.type이 Symbol(react.lazy)?
  → LazyComponent Fiber 생성
  → 로딩 완료 시 실제 컴포넌트로 교체
```

## 5. React 18+과의 대응 개념

| React 16/17                       | React 18+                  | 변화 요약                      |
| --------------------------------- | -------------------------- | ------------------------------ |
| Host 컴포넌트                     | 동일하게 유지              | 변화 없음                      |
| Custom 컴포넌트 (함수형/클래스형) | 동일하게 유지              | 변화 없음                      |
| `Fragment`, `Lazy`, `Memo`        | 동일하게 유지              | 변화 없음                      |
| 없음                              | **`Offscreen`** (실험적)   | 화면 밖 콘텐츠 숨기기·보존     |
| `Suspense` (실험적)               | `Suspense` (정식 지원)     | SSR + Concurrent Suspense 완성 |
| `React.lazy` + `Suspense`         | **`use()` 훅** (React 18+) | Promise 직접 소비 가능         |

## 6. 전체 개념 정리

```
[React 컴포넌트 분류]

                    컴포넌트
                    (props → 엘리먼트 반환)
                        │
          ┌─────────────┼──────────────┐
          │             │              │
    Host 컴포넌트   Custom 컴포넌트   Static 컴포넌트
    (DOM 태그)      (개발자 정의)     (React 내장)
    div, button     함수형, 클래스형  Fragment, Lazy
    input, img      MyComponent       Suspense, Memo
          │             │              │
          ▼             ▼              ▼
    DOM 엘리먼트   컴포넌트 엘리먼트  특수 엘리먼트
    type: 'div'    type: MyComponent  type: Symbol(...)
          │             │
          ▼             ▼
    Reconciler →  컴포넌트 함수 호출 → 다시 엘리먼트 처리
    실제 DOM 생성
```

# React 18+ — 엘리먼트와 컴포넌트 종류, 그리고 새로운 패턴

## 1. React 엘리먼트 = 자바스크립트 객체

### 1.1 JSX는 객체 생성 문법 설탕이다

```jsx
// 개발자가 작성하는 JSX
const element = <button className="primary">클릭</button>;

// 바벨/SWC가 변환하는 코드 (React 17+ 새 JSX 변환)
import { jsx as _jsx } from 'react/jsx-runtime';
const element = _jsx('button', { className: 'primary', children: '클릭' });

// 결과 객체
{
  $$typeof: Symbol(react.element),
  type: 'button',
  props: { className: 'primary', children: '클릭' },
  key: null,
  ref: null,
}
```

> ⚠️ **React 17+ 새 JSX 변환**: React 17 이전에는 JSX가 `React.createElement`로 변환되었다.  
> 그래서 JSX를 사용하는 파일에 `import React from 'react'`가 필수였다.  
> React 17+에서는 `react/jsx-runtime`을 자동으로 import하므로 더 이상 명시적 import가 불필요하다.

### 1.2 `$$typeof` — 보안 마커

```javascript
$$typeof: Symbol(react.element);
```

`$$typeof`가 존재하는 이유: JSON은 Symbol을 직렬화할 수 없다.  
서버에서 악의적인 JSON이 React 엘리먼트처럼 주입되는 **XSS 공격을 방지**한다.

```javascript
// JSON 파싱 결과로는 Symbol을 만들 수 없음
JSON.parse('{"$$typeof": "Symbol(react.element)"}');
// → { $$typeof: "Symbol(react.element)" } (문자열, Symbol 아님)
// → React가 유효한 엘리먼트로 인식하지 않음
```

## 2. 컴포넌트 세 가지 분류와 실전 코드

### 2.1 Host 컴포넌트 — DOM 태그

```jsx
// type이 문자열인 엘리먼트
<div className="container">    // { type: 'div', ... }
<button onClick={fn}>          // { type: 'button', ... }
<input type="text" />          // { type: 'input', ... }
<img src={url} alt="..." />    // { type: 'img', ... }
```

**Reconciler 처리:** 커밋 페이즈에서 `document.createElement('div')` 등으로 실제 DOM 노드를 생성한다.

### 2.2 Custom 컴포넌트 — 함수형·클래스형

```jsx
// 함수형 컴포넌트
function UserCard({ name, avatar }) {
  const [isExpanded, setIsExpanded] = useState(false);
  return (
    <div>
      <img src={avatar} />
      <span>{name}</span>
    </div>
  );
}
// { type: UserCard, props: { name: '...', avatar: '...' } }

// Reconciler 처리:
// renderWithHooks(current, wip, UserCard, props)
// → UserCard(props) 호출 → React 엘리먼트 반환 → 다시 처리
```

**클래스형 컴포넌트 (레거시):**

```jsx
class UserCard extends React.Component {
  render() {
    return <div>{this.props.name}</div>;
  }
}
// Reconciler 처리: new UserCard(props) → instance.render() 호출
```

### 2.3 Static 컴포넌트 — React 내장

```jsx
// Fragment: DOM 노드 없이 자식 묶기
<>
  <Header />
  <Main />
</>
// { type: Symbol(react.fragment), props: { children: [...] } }

// Suspense: 비동기 컴포넌트 로딩 대기
<Suspense fallback={<Spinner />}>
  <LazyComponent />
</Suspense>

// Memo: 불필요한 리렌더 방지
const MemoizedCard = React.memo(UserCard);

// Lazy: 코드 스플리팅
const LazyPage = React.lazy(() => import('./HeavyPage'));
```

## 3. React 18에서 추가·변경된 것들

### 3.1 `Suspense` — 완전히 지원되는 Concurrent 기능

React 16/17에서 Suspense는 `React.lazy`와만 함께 사용할 수 있었다.  
React 18에서는 **데이터 패칭**에도 Suspense를 사용할 수 있다.

```jsx
// React 18: 데이터 패칭 + Suspense
function UserProfile({ userId }) {
  // use() 훅으로 Promise 직접 소비
  const user = use(fetchUser(userId)); // Promise가 resolve될 때까지 Suspend
  return <div>{user.name}</div>;
}

function App() {
  return (
    <Suspense fallback={<Skeleton />}>
      <UserProfile userId={1} />
    </Suspense>
  );
}
```

### 3.2 `use()` 훅 (React 18.3+/19)

`use()`는 컴포넌트 내에서 Promise나 Context를 직접 소비하는 훅이다.

```jsx
import { use } from "react";

function UserCard({ userPromise }) {
  const user = use(userPromise); // Promise resolve까지 Suspend
  return <div>{user.name}</div>;
}
```

기존 훅과의 차이:

| 항목           | `useState`, `useEffect` 등 | `use()`                  |
| -------------- | -------------------------- | ------------------------ |
| 조건문 내 사용 | ❌ 불가                    | ✅ 가능                  |
| Promise 소비   | ❌ 불가                    | ✅ 가능                  |
| Context 소비   | `useContext` 별도 사용     | ✅ `use(MyContext)` 가능 |

### 3.3 `Offscreen` 컴포넌트 (실험적)

화면에서 숨겨도 상태를 보존하는 컴포넌트이다. `display: none`과 달리 React 상태가 살아있다.

```jsx
// 실험적 API (React 18 내부에서 활용)
<Offscreen mode="hidden">
  <HeavyComponent /> {/* 숨겨도 상태 유지 */}
</Offscreen>
```

현재 `React.unstable_Activity`라는 이름으로 일부 공개되어 있다.

## 4. JSX 변환 방식 변화 (React 16 → React 18)

### 4.1 React 16 방식 (구 변환)

```jsx
// JSX
const el = <div className="box">Hello</div>;

// 변환 결과
const el = React.createElement("div", { className: "box" }, "Hello");
// → 파일 상단에 import React from 'react' 필수
```

### 4.2 React 17+ 방식 (새 변환)

```jsx
// JSX (동일)
const el = <div className="box">Hello</div>;

// 변환 결과
import { jsx as _jsx } from "react/jsx-runtime"; // 자동 추가
const el = _jsx("div", { className: "box", children: "Hello" });
// → import React 불필요
```

| 차이점        | 구 변환       | 새 변환                   |
| ------------- | ------------- | ------------------------- |
| React import  | 필수          | 불필요                    |
| 번들 크기     | 상대적으로 큼 | 약간 작음                 |
| children 처리 | 나머지 인자   | `props.children`으로 통합 |

## 5. 컴포넌트 타입별 Fiber `tag` 값

Reconciler는 `element.type`을 보고 Fiber 노드의 `tag`를 결정한다.

| 컴포넌트 종류     | `element.type`           | Fiber `tag`         | 숫자 |
| ----------------- | ------------------------ | ------------------- | ---- |
| 함수형 컴포넌트   | 함수                     | `FunctionComponent` | 0    |
| 클래스형 컴포넌트 | 클래스                   | `ClassComponent`    | 1    |
| Fiber 루트        | -                        | `HostRoot`          | 3    |
| Host 컴포넌트     | 문자열                   | `HostComponent`     | 5    |
| Host 텍스트       | -                        | `HostText`          | 6    |
| Fragment          | `Symbol(react.fragment)` | `Fragment`          | 7    |
| Suspense          | `Symbol(react.suspense)` | `SuspenseComponent` | 13   |
| Memo              | `Symbol(react.memo)`     | `MemoComponent`     | 14   |
| Lazy              | `Symbol(react.lazy)`     | `LazyComponent`     | 16   |

## 6. 전체 정리

```
[React 18 컴포넌트 → 엘리먼트 → Fiber 흐름]

JSX 작성
  <UserCard name="Alice" />
        │ jsx-runtime 변환
        ▼
React 엘리먼트 생성
  { type: UserCard, props: { name: 'Alice' }, $$typeof: Symbol }
        │ Reconciler 처리
        ▼
Fiber 노드 생성 (tag: FunctionComponent)
        │ renderWithHooks
        ▼
UserCard({ name: 'Alice' }) 호출
  → <div>{name}</div> 반환
  → { type: 'div', props: { children: 'Alice' } }
        │
        ▼
Fiber 노드 생성 (tag: HostComponent)
        │ 커밋 페이즈
        ▼
document.createElement('div') → DOM 노드 생성
```

### 핵심 체크리스트

- [ ] React 엘리먼트는 불변 JS 객체이며 `type`, `props`, `key`, `ref` 필드를 가짐을 이해했다.
- [ ] `type`이 문자열이면 Host 컴포넌트, 함수/클래스면 Custom 컴포넌트, Symbol이면 Static 컴포넌트임을 이해했다.
- [ ] Host 컴포넌트는 Reconciler가 직접 DOM 노드를 생성하고, Custom은 함수를 호출하여 재귀적으로 처리함을 이해했다.
- [ ] `$$typeof: Symbol`이 XSS 공격 방지를 위한 보안 마커임을 이해했다.
- [ ] React 17+에서 `import React`가 불필요해진 이유(새 JSX 변환)를 이해했다.
- [ ] React 18에서 Suspense가 데이터 패칭까지 공식 지원함을 이해했다.
