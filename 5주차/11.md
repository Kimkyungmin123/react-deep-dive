# React Reconciler – Work 스케줄링 사전 작업 (React 16 기준)

📌 React Reconciler가 Work를 스케줄링하기 위한 사전 작업은 무엇인가?

Reconciler는 업데이트가 발생하면 expirationTime을 계산하고, 해당 컴포넌트의 Fiber에서 시작하여 VDOM 루트까지 올라가며 expirationTime을 기록하는 사전 작업을 수행한다.
이 과정에서 업데이트 우선순위 정보를 루트까지 전달하여 Scheduler가 Work를 스케줄링할 수 있도록 한다.

---

### 1. Reconciler의 Work 스케줄링 사전 작업

React에서 상태 변경이나 이벤트가 발생하면 다음과 같은 흐름으로 스케줄링이 준비된다.

```
update 발생
↓
expirationTime 계산
↓
fiber.expirationTime 설정
↓
parent.childExpirationTime 업데이트
↓
HostRoot까지 propagation
↓
root 반환
↓
Scheduler에 작업 전달
```

즉 Reconciler는 업데이트 우선순위를 계산하고 이를 루트까지 전달하는 역할을 수행한다.

### 2. expirationTime

💡 expirationTime이란?

expirationTime은 업데이트의 우선순위를 나타내는 값이다.

React는 이 값을 통해 여러 업데이트 중 어떤 작업을 먼저 처리할지 결정한다.

특징

- 이벤트 발생 시점을 기반으로 계산된다
- 숫자로 표현되는 priority 값이다
- 값이 작을수록 높은 우선순위

예시

```
Sync = 1
Task = 100
Idle = 1073741823
```

즉

```
작은 숫자 → 높은 우선순위
큰 숫자 → 낮은 우선순위
```

### 3. expirationTime이 저장되는 위치

React 16에서는 expirationTime이 Fiber 노드에 직접 저장된다.

Fiber 구조

```
Fiber {
expirationTime
childExpirationTime
}
```

설명

| 필드                | 의미                           |
| ------------------- | ------------------------------ |
| expirationTime      | 현재 fiber의 업데이트 priority |
| childExpirationTime | subtree 업데이트 priority      |

### 4. markUpdateTimeFromFiberToRoot

업데이트가 발생하면 React는 markUpdateTimeFromFiberToRoot 함수를 호출한다.

이 함수의 역할

- 1️⃣ 현재 fiber에 expirationTime 설정
- 2️⃣ 부모 fiber의 childExpirationTime 업데이트
- 3️⃣ 루트까지 propagation
- 4️⃣ 루트 반환

코드 흐름

```
markUpdateTimeFromFiberToRoot(fiber, expirationTime)

1. fiber.expirationTime 설정
2. alternate fiber에도 동일하게 설정
3. 부모 fiber로 이동
4. parent.childExpirationTime 업데이트
5. HostRoot까지 반복
6. root 반환
```

### 5. Fiber → Root propagation

업데이트는 현재 Fiber에서 시작하여 루트까지 전파된다.

이 과정에서

```
fiber.expirationTime
parent.childExpirationTime
```

이 값들이 업데이트된다.

구조

```
HostRoot
└ App
    └ Component
        └ Button (update 발생)
```

흐름

```
Button.expirationTime
↓
Component.childExpirationTime
↓
App.childExpirationTime
↓
Root.childExpirationTime
```

이렇게 하면 React는 subtree에 업데이트가 있는지 빠르게 판단할 수 있다.

### 6. childExpirationTime의 역할

childExpirationTime은 자식 Fiber에서 발생한 업데이트 priority를 부모에게 전달하는 값이다.

목적

- subtree에 업데이트가 있는지 빠르게 확인
- 불필요한 reconciliation을 skip

예시

```
childExpirationTime < renderExpirationTime
```

이면 React는 해당 subtree를 skip한다.

### 7. VDOM 루트 탐색

Reconciler는 업데이트가 발생한 Fiber에서 시작하여 HostRoot를 찾을 때까지 부모로 올라간다.

코드 개념

```
node = fiber

while (node.return !== null) {
node = node.return
}

if (node.tag === HostRoot) {
root = node.stateNode
}
```

이렇게 최종적으로 FiberRoot를 반환한다.

### 8. alternate Fiber

React는 double buffering 구조를 사용한다.

구조

```
current Fiber
↕ alternate
workInProgress Fiber
```

설명

| Fiber          | 의미                     |
| -------------- | ------------------------ |
| current        | 현재 화면에 반영된 Fiber |
| workInProgress | 작업 중인 Fiber          |

업데이트 시 두 Fiber 모두 expirationTime을 업데이트해야 한다.

### 9. React 16 스케줄링 관련 주요 함수

| 함수           | 역할                                                  |
| -------------- | ----------------------------------------------------- |
| scheduleWork   | 업데이트 스케줄링 시작                                |
| markUpdateTime | FromFiberToRoot expirationTime을 루트까지 propagation |
| performWork    | 실제 렌더 작업 수행                                   |

### 10. React 16 → React 18 변화

React 18에서는 expirationTime 기반 시스템이 제거되고
lane 기반 스케줄링 시스템으로 변경되었다.

개념 대응표
React 16| React 18
-|-
expirationTime| lanes
childExpirationTime| childLanes
scheduleWork |scheduleUpdateOnFiber
markUpdateTimeFromFiberToRoot| markUpdateLaneFromFiberToRoot

#### Fiber 구조 변화

React 16

```
Fiber {
expirationTime
childExpirationTime
}
```

React 18

```
Fiber {
lanes
childLanes
}
```

#### Lane 시스템 특징

lane은 bitmask 기반 priority 시스템이다.

예시

```
SyncLane
InputContinuousLane
DefaultLane
TransitionLane
IdleLane
```

lane 예시

```
0000001 Sync
0000010 Input
0000100 Default
```

장점

- 여러 업데이트 동시 관리 가능
- starvation 문제 해결
- Concurrent rendering 지원

### 11. 핵심 정리

React 16

```
expirationTime 기반 스케줄링
```

React 18+

```
lane 기반 스케줄링
```

업데이트 흐름 (React 16)

```
update 발생
↓
expirationTime 계산
↓
fiber.expirationTime 설정
↓
parent.childExpirationTime 업데이트
↓
HostRoot 탐색
↓
root 반환
↓
Scheduler 실행
```

## 한 줄 요약

React 16에서 Reconciler는 expirationTime을 계산하고 이를 Fiber에서 Root까지 전파하여 Scheduler가 Work의 우선순위를 판단할 수 있도록 한다.
