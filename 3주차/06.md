## 1. 기본 필기

### WorkInProgressHook

workInProgressHook(작업중인 hook)이 null ? 첫번째 hook 아니면 그다음 hook 으로 추가

(fiber.memoizedState 에 firstWorkInProgressHook 할당 → 컴포넌트와 hook(linked list)연결)

workInProgressHook은 tail pointer

### mountState

hook.memoizedState에 initialState 할당

- `const hook = mountWorkInProgressHook()`
  - hook 하나의 노드들이,
    hook1, hook2, hook3… 이렇게 연결되서 linkedList로 저장되어 관리됨

```tsx
function mountWorkInProgressHook(): Hook {
  const hook: Hook = {
    memoizedState: null,

    baseState: null,
    baseQueue: null,
    queue: null,

    next: null,
  };

  if (workInProgressHook === null) {
    // This is the first hook in the list
    currentlyRenderingFiber.memoizedState = workInProgressHook = hook;
  } else {
    // Append to the end of the list
    workInProgressHook = workInProgressHook.next = hook;
  }
  return workInProgressHook;
}
```

- hook 프로퍼티 값들 설명
  - memoizedState
    - hook객체, 즉 useState라는 hook을 호출해서 생성하는 hook 객체 안에는 관련된 state값을 가지고 있다.
    - 최종적으로 계산된 결과, 우리가 알고 있어야 하는 state값을 memoizedState키값으로 저장
  - baseState
  - queue
    - 하나의 hook 객체는 하나의 queue 객체를 가지고 있다.
  - next
    - linkedList
    - 다음 훅이 할당될 키 값
    - `hook` 객체는 linkedlist로 저장, linkedlist로 연결되어있음
- 리턴값 `wortInProgressHook`
  - 생성한 hook 객체 할당해서 리턴
- `if(workInProgressHook === null)`
  - true
    - 어떠한 hook도 작업중인 hook이 없다. → 첫번째 hook 할당
    - firstWorkInProgressHook이라는 전역변수에 workInProgressHook을 할당하고 , 그것에 hook 할당
  - false
    - 기존에 작업중인 hook 있다.
    - 새로운 hook 추가 → 리스트에 추가하는 방법
      - hook 이 저장되는 방법
        : workInProgressHook에도 next라는 프로퍼티가 있고, 연결하고 싶은 hook을 할당한다.
